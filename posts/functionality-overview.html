Operating System: Oasis Shell

Technical Specification & Functional Overview • April 2026

Oasis Shell is a high-performance system orchestrator and desktop layer designed to bridge the gap between traditional OS kernels and AI-native workflows. This document outlines the current working functionality and the underlying technology stack.

The Technology Stack

Kernel / Backend Rust (Tauri Framework)
Frontend UI React 19 + TypeScript
Visual Engine Three.js + React Force Graph 3D
Motion Design Framer Motion + GSAP
Audio Engine Web Audio API (Procedural Synthesis)
Styling Tailwind CSS (Vanilla-integrated)

Core Working Functionality

1. Neural Identity & Secure Handshake A deep-cyber boot sequence that validates user presence through a cryptographic handshake with the Rust kernel.
2. System Diagnostic Intelligence Real-time monitoring of CPU, RAM, and Storage directly via Rust's sysinfo. The shell provides a "Reality Anchor" for system health.
3. 3D Neural Cortex (Graph View) A force-directed 3D graph representation of the system's "Context Crates" and relational knowledge nodes.
4. Sentinel Vault (Zero-Knowledge) A secure archival layer for sensitive assets, utilizing PBKDF2-HMAC-SHA256 for neural key derivation within the Rust enclave.
5. Sensory Integration (Soundscape) Procedural, non-intrusive auditory feedback that adjusts based on system load and user activity levels.
6. Command Surface & Intent Parser A unified command palette that routes natural language intent to specific system actions and internal scripts.

Current Metrics

The system is currently operating in Alpha Stage (v0.1.0). All system calls are routed through an asynchronous IPC bridge to ensure the frontend remains performant even during heavy system load.
